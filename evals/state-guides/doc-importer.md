# doc-importer 状态定义

本文件只服务于评测与验收。这里允许从 Codex 侧通过 MCP 采集远端状态，但不反向约束 Skill.md 的工具口径。

## original_state

- 本次导入目标标题对应的笔记是否已存在
- 目标标题的同标题或近似标题命中数
- 无关控制笔记的完整快照
- 导入源 fixture 的基础指纹，例如文档标题、关键段落、预期图片数
- 如果是重复导入 case，还要记录既有目标笔记的完整快照

## expected_state

- 成功主路径：
  - 创建目标笔记，或按用户明确策略更新既有目标笔记
  - 标题正确
  - 来源说明落库
  - 关键段落和图片数与 fixture 假设一致
  - 无关控制笔记保持不变
- 幂等路径：
  - 重复运行不新增重复笔记
  - 如果用户明确要求“已有就不要再建”，原目标笔记也不应被无端改写
- 失败路径：
  - 不创建空白笔记或半成品笔记
  - 控制笔记不受污染
  - 最终回答明确说明无法完成导入

## final_state

- 通过与 `original_state` 相同的一组 collector 回读：
  - 实际目标笔记快照
  - 标题、来源说明、关键 XML、关键段落、图片数量
  - 同标题目标笔记的最终计数
  - 无关控制笔记是否保持不变
  - 必要时读取最终回答，用于判断是否清楚声明失败或降级
